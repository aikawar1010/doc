1. githubpagesの作成

構築環境：Windows10+Sphinx+markdown

1.1. Sphinxのインストール

以下の手順をベースに実施

https://www.kimoton.com/entry/20181009/1539049414

下記のようなエラーが出る場合

C:\Sphinx\source\index.rst:9: WARNING: toctree contains reference to document 'test/main' that doesn't have a title: no link will be generated

conf.py末尾に以下を挿入

def setup(app):
    app.add_config_value('recommonmark_config', {
            'auto_toc_tree_section': 'Welcome',
            }, 'env')
    app.add_transform(AutoStructify)

1.3. githubpagesへの公開

作成したページをgithub上に公開する。 githubは全く触れたことのない初心者であったため、リポジトリの作成方法などはググって出てきた適当なブログ記事参照。 新規に作成したリポジトリdoc（以後、doc）にビルドしたhtml配下のファイルを全て追加。

Github on Desktopを利用して、ローカルのhtml配下にdocをクローン。今後はSphinxのmakeで更新した情報をGithub on Desktopからコミットする形でメンテ。 現状本ページの作成の流れが以下のようになる。 1.Sphinx・Markdownによるページの作成 2.コンソール（自分の場合はGit bash）からmake実行 3.Github on Desktopからコミット 4.ページ更新確認 今後はgithubActionを利用してコミット自動化を検討したい。
